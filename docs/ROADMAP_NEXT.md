## 2. Vercel staging 배포

- **목적**: 실제 도메인 환경에서 지도/환경변수/SSG 검증
- **작업 내용**:
  - `docs/DEPLOYMENT_GUIDE.md`대로 Vercel 프로젝트 생성
  - env 설정 + Kakao 도메인 등록
  - staging URL에서 QA 재확인
- **위험 요소**:
  - Kakao 도메인 미등록으로 지도 로딩 실패
  - env 누락으로 Supabase read 실패
- **완료 기준**:
  - staging에서 메인/상세/검색/지도 정상 동작

## 3. 홈페이지 URL / 예약 링크 / 전화번호 보강 구조 만들기

- **목적**: 사용자 행동(전화/예약/홈페이지) 연결률 개선
- **작업 내용**:
  - 데이터 소스/수집 방식 정리(공식 홈페이지/플랫폼/크롤링/수기 보강)
  - 검증 규칙(정상 URL, 중복, 리다이렉트) 정의
  - 보강 파이프라인(수기/스크립트) 설계
- **위험 요소**:
  - 잘못된 링크로 UX 저하
  - 데이터 출처/정책 이슈
- **완료 기준**:
  - 링크 3종(homepage/booking/phone) 품질 기준/검증 방법 확정
